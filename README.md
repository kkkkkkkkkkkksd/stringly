# Stringly — сервис управления переводами (фронтенд)

Веб-приложение (TMS): таблица «ключ × язык» с разделами, доступ по API-токену.
Сейчас этап разработки фронта; бэка нет — данные отдаёт мок-сеть (MSW) прямо в браузере.

> **Вся разработка идёт локально.** Сайт открывается в браузере по адресу
> `http://localhost:5173`. Реальный домен/хост не нужен — он покупается и подключается в
> самом конце. Подробности и план — в папке [`docs/`](./docs/README.md) и в `CLAUDE.md`.

## 1. Что установить на Mac (один раз)

До этого веб-разработки не было — ставим два инструмента.

### Node.js (среда выполнения + npm)

1. Открой сайт **https://nodejs.org** и скачай версию **LTS** (кнопка слева).
2. Запусти скачанный файл `.pkg` и пройди установку (везде «Продолжить» / «Установить»).
3. Проверь, что установилось: открой приложение **Терминал** (Spotlight → «Терминал») и введи:

   ```bash
   node -v
   npm -v
   ```

   Должны напечататься номера версий (например `v22.x.x` и `10.x.x`). Если так — всё ок.

### VS Code (редактор кода)

1. Скачай с **https://code.visualstudio.com** и установи (перетащи в «Программы»).
2. Открой VS Code → File → Open Folder → выбери папку этого проекта (`localize-demo`).
3. Установи рекомендованные расширения (VS Code предложит сам, либо вкладка Extensions):
   - **ESLint** (`dbaeumer.vscode-eslint`) — подсветка ошибок.
   - **Prettier** (`esbenp.prettier-vscode`) — авто-форматирование.
   - **Tailwind CSS IntelliSense** (`bradlc.vscode-tailwindcss`) — подсказки классов.

   Чтобы код форматировался при сохранении: Settings (⌘,) → найди «Format On Save» →
   включи; и выбери Prettier как форматтер по умолчанию.

## 2. Запуск проекта

В VS Code открой встроенный терминал (Terminal → New Terminal) — он уже в папке проекта.

```bash
npm install      # 1) поставить зависимости (один раз, ~1–2 мин)
npm run dev      # 2) запустить — браузер откроется на http://localhost:5173
```

Остановить сервер — `Ctrl + C` в терминале. Чтобы снова запустить — опять `npm run dev`.

На стартовой странице внизу есть строка «Статус мок-API: ok (mock)» — это значит, что
связка React Query → HTTP-клиент → MSW работает.

## 3. Команды (npm scripts)

| Команда | Что делает |
|---------|------------|
| `npm run dev` | Локальный сервер с горячей перезагрузкой (`http://localhost:5173`) |
| `npm run build` | Проверка типов + продакшн-сборка в `dist/` |
| `npm run preview` | Посмотреть собранную версию локально |
| `npm run lint` | Проверка кода ESLint (в т.ч. правила слоёв) |
| `npm run format` | Авто-форматирование Prettier |
| `npm run typecheck` | Только проверка типов TypeScript |
| `npm test` | Юнит-тесты (Vitest) |
| `npm run mock:init` | Перегенерировать сервис-воркер MSW в `public/` (обычно не нужно) |

## 4. Стек и структура

React 18 + TypeScript · Vite · react-router · TanStack Query · Zustand ·
React Hook Form + Zod · Tailwind · MSW · Vitest.

```
src/
├── app/        инициализация: провайдеры, роутер
├── pages/      экраны (тонкий слой-композиция)
├── features/   функциональные блоки (появятся по мере разработки)
├── entities/   доменные сущности
├── shared/     переиспользуемое: api (httpClient, ключи кэша), config, ui (токены)
└── mocks/      MSW: мок-сеть вместо бэка
```

Правило зависимостей: `app → pages → features → entities → shared` (импорт только «вниз»).
Подробно — в `docs/04-architecture.md`. Дизайн-токены (цвета/шрифты) — `docs/14-design-system.md`.

## 5. Окружение (.env)

```
VITE_API_MODE=mock        # mock = данные из MSW в браузере; real = реальный бэк (позже)
VITE_API_BASE_URL=/api/v1 # относительный путь — работает и локально, и на проде
```

Файл `.env` уже создан для локальной работы. Менять обычно не нужно до подключения бэка.
