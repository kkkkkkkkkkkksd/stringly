# 03. Спецификация экранов

Для каждого экрана: назначение, состав, какие данные тянет (queries), какие действия
(mutations) и какие технологии задействованы.

---

## 1. Лендинг `/`

**Назначение:** объяснить продукт и привести к регистрации/входу.

**Состав:**
- Хедер: логотип Stringly слева, кнопка **«Войти»** справа (+ «Регистрация»).
- Hero-блок: заголовок, краткое описание, CTA «Начать бесплатно».
- 3–4 блока с фичами (единая таблица, API-доступ, много языков).
- Футер.

**Данные:** нет (статика). **Технологии:** React, react-router (`<Link>`), CSS-модули/Tailwind.

**Доступ:** публичный. Если пользователь уже залогинен — кнопка ведёт сразу в `/app`.

---

## 2. Регистрация `/register`

**Назначение:** создать аккаунт.

**Поля:** email, пароль, повтор пароля (+ опц. имя).

**Валидация (Zod):** корректный email; пароль ≥ 8 символов, буквы+цифры; пароли совпадают.

**Действие:** `mutation registerUser` → при успехе сохраняем токены, редирект в `/app`.

**Технологии:** React Hook Form + Zod (валидация), React Query (mutation), react-router.

**Состояния:** loading на кнопке, ошибки полей, серверная ошибка (email занят).

---

## 3. Вход `/login`

**Поля:** email, пароль. **Действие:** `mutation login` → access-токен → редирект.

**Технологии:** те же, что в регистрации. Ссылка «Забыли пароль?» (v2).

> **Сессия (упрощённо на первое время):** один **бессрочный access-токен**, без
> refresh-механики. Хранится в Zustand + localStorage. Refresh/ротацию токенов добавим позже.

---

## 3.5. Онбординг: создание первого проекта `/onboarding`

**Назначение:** у пользователя без проектов нет рабочей области — просим создать первый.

**Когда:** после регистрации/входа, если у аккаунта 0 проектов → редирект сюда.

**Состав:** одно поле **«Название проекта»** + кнопка «Создать». Минимум трения.

**Действие:** `mutation createProject({ name })` → проект становится активным → редирект в `/app/table`.

**Технологии:** RHF + Zod, React Query. **Доступ:** protected.

---

## 4. Каркас приложения `/app/*` (layout)

**Назначение:** общий каркас для защищённых страниц.

**Состав:**
- **Левая панель (Sidebar):**
  - **Переключатель проекта** (вверху): UI-**заглушка** на первое время (показывает
    текущий проект, выбор/переключение пока неактивны). Кнопка **«+ Проект»** тоже
    заглушка (disabled / «скоро»). Архитектурно настройки и данные остаются привязаны к
    активному проекту (`pid`), чтобы включить переключение позже без переделок.
  - Пункты: **Таблица** (`/app/table`), **Настройки** (`/app/settings`).
  - Внизу: профиль + «Выйти».
- **Контентная область** — `<Outlet/>` из react-router.

**Гарды:** `ProtectedRoute` — нет сессии → `/login`. `RequireProject` — у аккаунта 0
проектов → `/onboarding`.

**Активный проект:** хранится в Zustand (`activeProjectId`) + в URL/localStorage, чтобы
переживал перезагрузку. Все запросы зависят от него (`pid` в query keys).

**Технологии:** react-router (nested routes + `Outlet`), Zustand (активный проект, auth).

---

## 5. Таблица переводов `/app/table` ⭐ ядро продукта

**Назначение:** просмотр и редактирование переводов.

**Состав:**
- **Вкладки разделов (namespaces)** сверху: `strings`, `plurals`, `login_strings`,
  `region_strings`, `widget_strings`, `profile_strings`, `esim_strings`,
  `installation_strings`, `loyalty_program`, `rate_app` … — соответствуют референсу.
  При создании раздела выбирается **тип**: «Обычные строки» (`strings`) или «Plurals».
  От типа зависит вид ячейки-редактора (см. экран 5, плюрализация).
- **Тулбар:** кнопки **«+ Ключ»**, **«+ Язык»** (быстрый доступ — открывает добавление
  языка прямо из таблицы, без захода в настройки), **«Сохранить»** (активна при наличии
  изменений, показывает счётчик). Поиск и фильтр («только пустые», по языку) **в этой
  версии неактивны** (UI присутствует, но задизейблен — реализация в следующей итерации).
- **Таблица:** колонки `code` (ключ), `comment`, далее по колонке на каждый язык
  (`en`, `ru`, `ar`, `cs`, `da`, `nl`, `fi`, `fr`, `de`, `hu`…). **Языковые колонки
  формируются автоматически из списка языков проекта** — одинаковы для всех разделов,
  отдельно в каждом разделе язык добавлять не нужно. Левые колонки закреплены (sticky),
  горизонтальный скролл по языкам.
- **Инлайн-редактирование** ячеек; изменённые ячейки помечаются (dirty), пока не сохранены.

**Данные (queries):**
- `getNamespaces(projectId)` — список вкладок.
- `getLanguages(projectId)` — список языков проекта = колонки (общие для всех разделов).
- `getTranslations(projectId, namespaceId, { page })` — строки (пагинация/инфинит-скролл).
  Параметры `search`/`filter` заложены в контракт, но в этой версии не используются.

**Действия (mutations):**
- `addKey`, `updateTranslations(batch)` — батч dirty-ячеек, `deleteKey`. Оптимистичные
  апдейты + инвалидация кэша React Query.
- `addLanguage` доступен и из быстрого доступа в таблице, и из настроек языков (6.1) —
  это один и тот же централизованный список; после добавления колонка появляется во всех
  разделах автоматически.
- `createNamespace({ name, type })` — `type` = `strings | plurals`.

**Технологии:** **TanStack Table** (модель колонок/строк) + **TanStack Virtual**
(виртуализация), React Query (данные+мутации), Zustand (буфер несохранённых правок).
Подробно — в `08-table-design.md`.

## 6. Настройки `/app/settings`

Вложенные вкладки (nested routes):

### 6.1 Языки `/app/settings/languages` ⭐ (MVP, дефолтная вкладка)
- **Централизованное** управление языками проекта: список языков, базовый язык, флаг RTL.
- Кнопка **«+ Язык»** (выбор ISO-локали). Добавленный язык = новая колонка **во всех
  разделах** сразу.
- Доступно также из **быстрого доступа в таблице** (см. экран 5) — чтобы не заходить
  каждый раз в настройки.
- Редактирование/удаление языков — admin.

**Данные:** `getLanguages(projectId)`; **Действия:** `addLanguage`, `removeLanguage`, `setBaseLanguage`.

### 6.2 Участники `/app/settings/members` ⭐ (MVP)
- Таблица участников проекта: имя/email, **роль** (Admin / Translator / Viewer), дата добавления.
- **Своя строка помечена** («вы») + бейдж своей роли (роль также видна в сайдбаре).
- **Виден всем** участникам проекта (read-only для не-админов).
- **Только admin** может менять роли / добавлять / удалять (в MVP — UI с правами, само
  приглашение может быть заглушкой; полноценное — v2, F-27).
- Admin — это владелец, создавший проект.

**Данные:** `getMembers(projectId)`; **Действия (admin):** `updateMemberRole`, `removeMember`.

### 6.3 Подключения `/app/settings/connections` (later — заглушка)
Выпуск API-токенов делаем позже. В MVP вкладка присутствует как **заглушка**
(«скоро»). Запланированное наполнение:
- Список выпущенных **API-токенов**: имя, маскированный токен, дата создания, последнее использование.
- Кнопка «Создать токен» → модал → токен показывается один раз.
- Кнопка «Отозвать», сниппет-пример вызова API.

### 6.4 Профиль / аккаунт `/app/settings/profile` (v2)
- Смена email, смена пароля, имя, информация об аккаунте (дата регистрации). «Выйти из всех сессий» — поздн.

### 6.5 Проект `/app/settings/project` (v2)
- Переименование проекта, базовый язык (тот же централизованный, что в «Языках»), slug,
  формат экспорта по умолчанию. «Опасная зона»: удаление проекта.

### 6.6 Предпочтения `/app/settings/preferences` (MVP — есть)
- Тема оформления: light / dark / system (готово). Язык интерфейса самого Stringly —
  **пока только русский** (мультиязычность интерфейса — later). Плотность таблицы — позже.

### 6.7 Документация `/app/settings/documentation` (заглушка, MVP → later)
- В MVP — вкладка-заглушка («скоро»); состав наполнения определим позже.

### 6.8 История изменений `/app/settings/history` (заглушка, MVP → v2, F-17)
- Журнал активности: кто/что/когда менял (ключи, переводы, языки, участники),
  фильтры по пользователю/дате. В MVP — **заглушка** (вкладка/кнопка присутствует).

**Технологии:** react-router (nested), React Query, React Hook Form (модалы),
ролевые гарды (см. `11-roles-and-permissions.md`).

## Карта маршрутов

```
/                         Лендинг (public)
/login                    Вход (public)
/register                 Регистрация (public)
/onboarding               Создание первого проекта (protected, если 0 проектов)
/app                      Layout (protected + RequireProject)
  /app/table              Таблица переводов  ← дефолтный редирект из /app
  /app/settings           Layout настроек
    /app/settings/languages     Языки проекта, централизованно (MVP, дефолт)
    /app/settings/members       Участники + роли (MVP)
    /app/settings/preferences   Тема + (поздн.) интерфейс/плотность (MVP — есть)
    /app/settings/connections   Токены (later — заглушка)
    /app/settings/documentation Документация (заглушка, later)
    /app/settings/history       Журнал изменений (заглушка, MVP → v2)
    /app/settings/profile       Профиль / аккаунт (v2)
    /app/settings/project       Проект (v2)
*                         404
```
