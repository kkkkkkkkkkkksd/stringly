# Stringly — документация проекта

**Stringly** — веб-сервис управления переводами (TMS): централизованное хранение строк
локализации в БД, удобная таблица «ключ × язык» с разделами, выдача переводов клиентским
приложениям по API-токену.

> Рабочее название продукта — **Stringly** (обоснование и альтернативы в `01`).
> Имя меняется в одном конфиге бренда и не влияет на архитектуру.

## Статус

Этап проектирования. Эта папка содержит **только документацию** (анализ, архитектура,
план). Кода ещё нет — он пишется по этому плану. Бэкенд будет позже; фронт проектируется
под контракт API и до его появления работает на mock-слое (MSW).

## Ключевые решения (TL;DR)

- **Стек:** React + TypeScript + Vite, react-router v6, **React Query** (серверные
  данные), **Zustand** (клиентское состояние), React Hook Form + Zod (формы/валидация),
  Tailwind. Обоснование — `05`.
- **Таблица:** **TanStack Table + TanStack Virtual** — headless-модель + виртуализация
  строк/колонок для стабильной работы на тысячах строк. Запасной вариант — AG Grid. См. `08`.
- **Данные:** всё через React Query с централизованными query keys, серверной
  пагинацией и оптимистичными батч-правками. Поиск/фильтры заложены в контракт, но в
  этой версии UI неактивен. См. `07`.
- **Без бэка:** MSW мокает реальный API; переключение на боевой бэк — сменой флага
  `VITE_API_MODE`, без правок в фичах.
- **Масштабируемость:** Feature-Sliced структура, строгие правила зависимостей между
  слоями (форсятся линтером), типобезопасность end-to-end через Zod.

## Решено по итогам обсуждения (в MVP)

- **Сессия:** один **бессрочный токен**, без refresh — упрощение на первое время.
- **Проект (один):** онбординг создаёт первый проект по имени. Переключатель проектов и
  «+ Проект» — **заглушки**; данные/настройки привязаны к активному проекту, мультипроектность
  включим позже без переделок.
- **Языки — централизованно:** задаются один раз на уровне проекта, все разделы автоматически
  берут из них колонки; добавление доступно из настроек **и из быстрого доступа в таблице**.
- **Разделы с типом:** при создании раздела выбор «Обычные строки» или «Plurals».
- **Участники:** список с ролями (admin/translator/viewer) виден всем, редактирует только
  admin (он же создаёт проекты). Полноценное приглашение — v2. См. `11`.
- **Заглушки/следующая итерация:** API-токены («Подключения»), поиск и фильтры в таблице.
