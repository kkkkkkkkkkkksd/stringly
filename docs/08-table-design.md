# 08. Дизайн таблицы переводов (ядро продукта)

Самый нагруженный экран. Главная задача — **стабильная, плавная работа** при тысячах
ключей × десятках языков с инлайн-редактированием.

## Структура

- Левые **закреплённые (sticky) колонки**: `code` (ключ) и `comment`. Не уезжают при
  горизонтальном скролле.
- Далее по колонке на каждый язык: `en`, `ru`, `ar`, `cs`, `da`, `nl`, `fi`, `fr`,
  `de`, `hu`… (как на референсе). RTL-языки (`ar`, `he`) рендерят ячейку с `dir="rtl"`.
- **Вкладки разделов (namespaces)** над таблицей — переключение источника строк.

## Технология: TanStack Table + TanStack Virtual

**TanStack Table (headless)** даёт модель: определение колонок, сортировка, фильтрация,
закрепление колонок — но не диктует разметку. Мы сами рендерим строки/ячейки, что
позволяет встроить редакторы и виртуализацию.

**TanStack Virtual** рендерит только видимые строки. При 10 000 строк в DOM живёт ~30–50
строк вместо 10 000 → скролл и ввод остаются плавными.

> Почему не «обычная таблица»: map по тысячам строк создаёт десятки тысяч DOM-узлов,
> каждый ререндер дорогой, ввод в ячейку лагает. Виртуализация — обязательна.

### Виртуализация строк (и при необходимости — колонок)

```ts
const rowVirtualizer = useVirtualizer({
  count: rows.length,
  getScrollElement: () => scrollRef.current,
  estimateSize: () => 40,   // высота строки
  overscan: 10,
});
```

Если языков очень много (30+), включаем и **виртуализацию колонок** (горизонтальный
`useVirtualizer`), чтобы не рендерить невидимые языковые колонки.

## Инлайн-редактирование без тормозов

Ключевая проблема: правка одной ячейки не должна ререндерить всю таблицу.

Решения:
1. **Локальный буфер dirty-ячеек в Zustand** (`Map<keyId:langCode, value>`), отдельный
   от кэша React Query. Ячейка-редактор подписана только на свою запись (точечная
   подписка через селектор) → ререндерится только она.
2. **Мемоизация ячеек** (`React.memo`) — ячейка перерисовывается только при смене
   своего значения/статуса.
3. **Стабильные `columns` и `data`** (`useMemo`) — иначе TanStack Table пересоберёт модель.
4. **Сохранение — батчем:** все правки из буфера уходят одним `PATCH` по кнопке
   «Сохранить» (оптимистичный апдейт, см. `07-api-and-queries.md`).

### Индикация изменений

- dirty-ячейка подсвечивается (рамка/фон).
- кнопка «Сохранить» показывает счётчик: «Сохранить (12)».
- предупреждение при уходе со страницы с несохранёнными правками (`beforeunload` + router guard).

## Тулбар таблицы

- Поиск (дебаунс 300 мс, серверный).
- Фильтры: «только пустые», по статусу, по конкретному языку.
- **+ Язык** — модал: выбор ISO-локали → `addLanguage` → новая колонка.
- **+ Ключ** — модал: `code` + `comment` → `addKey` → новая строка (скролл к ней).
- **Сохранить** — батч dirty-ячеек.
- (v2) Импорт/экспорт, переключатель отображения пустых.

## Доступность и UX

- Навигация по ячейкам с клавиатуры (Tab/стрелки, Enter — подтвердить, Esc — отмена).
- Sticky-заголовок и sticky-первые колонки.
- Skeleton при загрузке страницы строк.
- Пустое состояние: «В этом разделе пока нет ключей» + кнопка «+ Ключ».

## Ориентиры производительности (цели)

| Метрика | Цель |
|---------|------|
| Рендер 10k строк × 12 языков | без фриза, скролл 60 fps |
| Задержка ввода в ячейку | < 50 мс |
| Переключение вкладки раздела | < 200 мс (с кэшем — мгновенно) |
| Сохранение 100 правок | один запрос, оптимистичный отклик |

## Запасной вариант

Если в ходе разработки TanStack+Virtual окажется недостаточно (например, понадобятся
группировка, заморозка диапазонов, Excel-подобные фичи) — рассматриваем **AG Grid** как
готовую грид-систему. Это задокументированный запасной путь, не меняющий слой данных
(api/React Query остаются те же).
