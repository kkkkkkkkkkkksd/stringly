# 01. Видение продукта и нейминг

## Проблема

Команды, которые делают мультиязычные продукты (мобильные приложения, веб, виджеты),
хранят переводы в Google Sheets, в `.json`/`.xml`/`.strings` файлах или в коде. Это
приводит к типичным болям:

- нет единого источника правды — переводы рассинхронизированы между платформами;
- переводчик правит таблицу, а разработчик вручную переносит строки в код;
- нет контроля версий, истории изменений и ревью;
- нельзя быстро найти, где используется ключ, и какие языки не заполнены;
- добавить новый язык = ручная работа по всему проекту.

## Решение

**TMS (Translation Management System)** — веб-сервис, где переводы хранятся
централизованно в БД, редактируются в удобной таблице (ключ × язык), разбиты на
смысловые разделы (namespaces) и отдаются клиентским приложениям через API по токену.

Это аналог Lokalise / Crowdin / Phrase / Tolgee, но сфокусированный и масштабируемый
под наш сценарий.

## Целевые пользователи (роли)

| Роль | Что делает |
|------|-----------|
| **Owner / Admin** | Создаёт проект, управляет участниками, выпускает API-токены |
| **Translator / Editor** | Заполняет и правит переводы в таблице |
| **Developer** | Интегрирует приложение через API-токен, тянет переводы |
| **Reviewer** (на будущее) | Подтверждает/отклоняет переводы |

## Ключевая ценность

1. Единая таблица переводов с делением на разделы.
2. Добавление новых языков и ключей в пару кликов.
3. API-доступ по токену — приложение всегда получает актуальные данные.
4. Масштабируемость: тысячи ключей × десятки языков без падения UI (виртуализация).

## Креативные названия сервиса

Подобраны варианты с разным характером. Рекомендация отмечена ⭐.

### Короткие и брендовые
- **Lingofy** — «делать многоязычным», легко произносится, свободный нейминг.
- ⭐ **Stringly** — «string» (строка) + игривый суффикс; точно отражает суть (строки переводов), хорошо ложится на домен `stringly.io`.
- **Polyloco** — poly (много) + loco/localization.
- **Verba** — лат. «слова», солидно и коротко.
- **Glossa** — греч. «язык», ассоциация с глоссарием.

### Описательно-технические
- **LangForge** — «кузница языков», подчёркивает инженерный масштаб.
- **LocaleHub** — хаб локалей, понятно международной аудитории.
- **KeyLingo** — «ключ + язык», прямо про модель ключ→перевод.
- **StringSmith** — «мастер строк».
- **i18nly** — для технической аудитории (i18n = internationalization).

### Рекомендация
**Stringly** как рабочее имя продукта: коротко, запоминается, точно по смыслу
(сервис про строки переводов), легко строить узнаваемость (`@stringly`, `stringly.io`).
В коде и документации можно использовать namespace `stringly` / `stl`.

> В остальной документации продукт условно называется **Stringly**. Имя меняется в одном
> месте (конфиг бренда), поэтому смена названия не влияет на архитектуру.
