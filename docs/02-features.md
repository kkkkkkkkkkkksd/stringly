# 02. Фичи

Список фич с приоритетами. **MVP** — обязательно для первой версии. **v2** — следующая
итерация. **later** — бэклог.

## Аутентификация и доступ

| # | Фича | Приоритет | Описание |
|---|------|-----------|----------|
| F-1 | Лендинг (стартовый экран) | MVP | Описание продукта + кнопка «Войти» сверху |
| F-2 | Регистрация (email + пароль) | MVP | Кастомная форма, валидация, подтверждение пароля |
| F-3 | Вход (email + пароль) | MVP | JWT access + refresh, хранение сессии |
| F-4 | Выход / refresh токена | MVP | Автообновление access-токена |
| F-5 | Восстановление пароля | v2 | По email |

## Версии и релизы (будущее)

| # | Фича | Приоритет | Описание |
|---|------|-----------|----------|
| F-30 | Версионирование / релизные каналы | later | Наборы переводов привязаны к каналу: **planned** (в работе) → **testing** (на тесте) → **released** (в релизе). Можно держать черновую версию отдельно от релизной, продвигать между каналами, сравнивать diff, откатываться. Снимок (snapshot) релиза по версии/тегу. Публичный API отдаёт перевод для выбранного канала/версии (напр. `?channel=released`). Дополняет аудит F-17, не заменяет его |
| F-31 | Сравнение и diff версий | later | Что изменилось между версиями/каналами: новые/изменённые/удалённые ключи, незаполненные языки перед релизом |

## Решения по рекомендациям (что делаем сразу)

По итогам обсуждения в MVP уходят:

- **Проект (один) + онбординг** (F-23): создаём первый проект по имени. Переключатель
  проектов и «+ Проект» (F-24, F-25) — **заглушки**; настройки/данные привязаны к
  активному проекту, чтобы включить мультипроектность позже без переделок.
- **Централизованные языки** (F-10, F-28): языки на уровне проекта, колонки во всех
  разделах автоматически; добавление доступно из настроек **и из быстрого доступа в таблице**.
- **Разделы с типом** (F-9): при создании раздела выбор «Обычные строки» или «Plurals».
- **Список участников с ролями** (F-26): виден всем, редактирует только admin.

Заглушки / следующая итерация:

- **Сессия:** один бессрочный токен, без refresh (упрощение на первое время).
- **Подключения / API-токены** (F-20): вкладка-заглушка, реализуем позже.
- **Поиск и фильтры** (F-14): UI неактивен, реализация — следующая итерация.

Остаётся на потом:

- **Дашборд проекта** — % заполненности по языкам, пустые ключи, последние изменения.
- **Глобальный поиск** по всем разделам сразу — v2.
- **Статусы перевода (F-29)** — workflow new / needs review / translated с авто-устареванием
  при изменении исходника; индикация в ячейке + фильтр по статусу. Поле `status` уже
  заложено в модель (06) — на Шагах 3–4 сознательно показывали только «пусто/заполнено».
- **Версионирование / релизные каналы (F-30, F-31)** — planned → testing → released,
  снимки релизов, diff между версиями, выбор канала в публичном API.
