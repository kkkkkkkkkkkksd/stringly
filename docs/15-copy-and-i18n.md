# 15. Ресурсы: тексты UI (i18n) и ассеты

> Группа `src/shared/resources` = `i18n` (тексты) + `assets` (иконки/изображения).
> Ниже — про тексты; правило для ассетов: иконки/картинки в `resources/assets`
> именованными компонентами, без «сырых» SVG-path в JSX.


> **Правило (обязательно):** ни одной видимой строки интерфейса не пишем прямо в
> компонентах. Все тексты живут в `src/shared/resources/i18n` и берутся оттуда через
> `texts.<namespace>.<key>`. Аналог `Localizable.strings` в iOS.

Зачем: единый источник правды по копирайтингу, лёгкая правка формулировок, готовность к
многоязычному интерфейсу (миграция на i18next без правок в компонентах), отсутствие
дублей.

> Не путать с **переводами-строками продукта** — теми, что пользователь редактирует в
> таблице (хранятся в БД/моках). Здесь — копирайтинг самого сервиса.

## Где лежит

```
src/shared/resources/i18n/
├── common.ts      # общие слова: действия (Сохранить, Скопировать, Войти…), состояния, статусы
├── landing.ts     # тексты лендинга
├── auth.ts        # вход/регистрация (включая сообщения валидации)
├── onboarding.ts  # создание первого проекта
├── app.ts         # каркас приложения, заглушки экранов, 404
└── index.ts       # export const texts = { common, landing, auth, onboarding, app }
```

Импорт всегда из барреля: `import { texts } from '@/shared/resources/i18n';`

Ассеты (иконки/изображения) — рядом, в `src/shared/resources/assets` (именованные
компоненты, без «сырых» SVG-path в JSX): `import { TableRowsIcon } from '@/shared/resources/assets';`

## Как использовать

```tsx
import { texts } from '@/shared/resources/i18n';
const t = texts.auth;            // namespace экрана

<h1>{t.tabs.login}</h1>
<Button>{texts.common.actions.save}</Button>
```

Сообщения валидации (Zod) — тоже из модуля:

```ts
const schema = z.object({ email: z.string().email(texts.auth.errors.email) });
```

## Соглашения

- **common** — то, что встречается на разных экранах (действия, состояния loading/empty/error,
  названия статусов). Не дублируй такие слова в namespace'ах экранов.
- **namespace экрана** (`landing`, `auth`, …) — тексты, специфичные для него.
- Ключи — по смыслу (`submit.login`, `errors.passwordMin`), не по случайному номеру.
- Файлы помечены `as const` → строгая типизация и автодополнение по ключам.
- Новый экран/фича → заводи свой namespace-файл и подключай в `index.ts`.

## На будущее (мультиязычный интерфейс)

Когда понадобится несколько языков UI: оборачиваем `texts` в i18next (или храним по
локалям `ru/`, `en/` с тем же деревом ключей) и заменяем `texts.x.y` на `t('x.y')`.
Компоненты уже обращаются к ключам, поэтому переход локальный — без переписывания экранов.
