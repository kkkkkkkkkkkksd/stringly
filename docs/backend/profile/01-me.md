# 03. Текущий пользователь — `GET api/v1/auth/me`

Возвращает профиль владельца токена. Нужен фронту для восстановления сессии после
перезагрузки страницы (токен лежит в localStorage) и для проверки, что токен ещё валиден.

**Поддерживает:** пункты ТЗ 1–2 (продолжение сессии), 7 (профиль пользователя с мультидоступом).

## Эндпоинт

| | |
|---|---|
| Метод | `GET` |
| URL | `api/v1/auth/me` |
| Авторизация | **обязательна** — `Authorization: Bearer <accessToken>` |

## Тело запроса

Нет (GET).

## Ответы

### 200 OK

```json
{
  "user": {
    "id": "usr_018f3c8e-7a21-7b3c-9f10-2c4d5e6f7a8b",
    "email": "kirill@acme.com",
    "createdAt": "2026-06-11T09:00:00Z"
  }
}
```

### Ошибки

| HTTP | code | Причина |
|------|------|---------|
| 401 | `UNAUTHENTICATED` | нет токена / истёк / отозван (`sessions.revoked_at`) |

## Бэкенд: логика

1. Извлечь токен из `Authorization`. Нет → `401`.
2. `sha256(token)` → найти `sessions` по `token_hash`, где `revoked_at IS NULL` и (`expires_at
   IS NULL OR expires_at > now()`). Не найдено → `401`.
3. Обновить `last_used_at` (можно асинхронно/батчем, чтобы не писать на каждый запрос).
4. Загрузить `users` по `session.user_id`. Вернуть `{ user }`.

Это поведение — общий middleware аутентификации для всех защищённых эндпоинтов (projects/*, settings/*, invitations/*).

## Фронтенд: логика

- **Слой:** `features/auth`.
- **API:** `authApi.me()` → `httpClient.get('/auth/me')` → парс `z.object({ user: userSchema })`.
- **Хук:** `useMe()` — `useQuery({ queryKey: qk.me(), queryFn: authApi.me, enabled: !!token })`
  (уже реализовано). Токен берётся из `sessionStore`.
- **Восстановление сессии:** `sessionStore` персистит токен в localStorage и при rehydrate
  отдаёт его в `httpClient` (`setAuthToken`). `useMe` подтверждает валидность.
- **На `401`:** `httpClient` инициирует глобальный разлогин (`sessionStore.clear()`) и редирект
  на `/login` (заложенный TODO в httpClient — здесь реализуется).

## Edge cases

- Токен валиден, но пользователь удалён → `401` (сессия каскадно удалена).
- Несколько вкладок: разлогин в одной чистит общий localStorage-ключ `stringly-session`.
