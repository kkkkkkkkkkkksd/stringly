# 12. Удалить участника — `DELETE api/v1/projects/{projectId}/members/{userId}`

Админ исключает участника из проекта. Также покрывает «выйти из проекта» (пользователь удаляет
сам себя).

**Поддерживает:** пункт ТЗ 3 (управление составом).

## Эндпоинт

| | |
|---|---|
| Метод | `DELETE` |
| URL | `api/v1/projects/{projectId}/members/{userId}` |
| Авторизация | обязательна — `Bearer`; `admin` (или сам участник для «выйти») |

## Path-параметры

| Параметр | Пример | Описание |
|----------|--------|----------|
| `projectId` | `prj_018f...aaa` | проект |
| `userId` | `usr_018f...222` | удаляемый участник (или `me`/свой id для выхода) |

## Ответы

### 204 No Content

Тело пустое.

### Ошибки

| HTTP | code | Причина |
|------|------|---------|
| 401 | `UNAUTHENTICATED` | нет токена |
| 403 | `FORBIDDEN` | не admin и не сам себя удаляет |
| 404 | `NOT_FOUND` | участник/проект не найден |
| 422 | `UNPROCESSABLE` (`OWNER_IMMUTABLE`) | нельзя удалить владельца проекта |
| 422 | `UNPROCESSABLE` (`LAST_ADMIN`) | удаление оставит проект без admin |

## Бэкенд: логика

1. Аутентификация → `currentUser`.
2. Разрешено, если `currentUser` — admin проекта **или** `currentUser.id == userId` (выход из проекта).
3. Найти `membership (project_id, user_id)`. Нет → `404`.
4. **Инварианты:**
   - Владельца удалить нельзя → `422 OWNER_IMMUTABLE` (сначала передать владение — future).
   - Нельзя удалить последнего admin → `422 LAST_ADMIN`.
5. `DELETE FROM memberships WHERE project_id=$1 AND user_id=$2`. Данные перевода остаются за
   проектом (членство — только доступ).
6. Вернуть `204`.

Безопасность: «выйти» нельзя, если ты единственный admin/владелец — сначала назначить другого.
Удаление — событие для аудит-лога.

## Фронтенд: логика

- **Слой:** `features/members`.
- **API:** `membersApi.remove(projectId, userId)` → `httpClient.delete(...)`.
- **Хук:** `useRemoveMember(projectId)` — `useMutation`; `onSuccess` → инвалидировать
  `qk.members(pid)`; если удалил себя → `qk.projects` + редирект (проект пропал из списка).
- **UI:** кнопка «Удалить» в строке (под `<Can perm="members:manage">`) + «Выйти из проекта»
  для себя; подтверждение перед удалением.
- **Состояния:** оптимистичное удаление строки; `422` → тост («нельзя удалить владельца/последнего admin»).

## Edge cases

- Самоудаление последнего admin → `LAST_ADMIN` (UI предлагает сначала назначить нового admin).
- Повторное удаление (строки уже нет) → `404`, тихо игнорировать в UI.
