# 16. Фидбэк: ошибки и успехи (конвенция)

Единые правила, **где и как** показывать ошибки и успех, чтобы не было рассинхрона между
экранами. Все каналы используют токены дизайн-системы (`14`) и готовые компоненты —
не верстаем фидбэк руками.

> TL;DR: **валидация поля → `Input error`; результат отправки формы → `FormNote` (инлайн);
> действие-кнопка без формы → тост; ошибка загрузки данных с собственным UI → `EmptyState`;
> подтверждение опасного действия → `ConfirmDialog`; «скоро» → `SoonCard`; краш → `ErrorBoundary`.**

## Каналы фидбэка (когда что)

| Ситуация | Канал | Компонент / механизм |
|----------|-------|----------------------|
| Валидация поля (клиент, Zod/RHF) | инлайн под полем | `Input error={...}` |
| Результат отправки **формы** (успех/ошибка сервера) | инлайн под формой | `FormNote` (`shared/ui`) |
| Результат **действия-кнопки** без формы (списки/таблица) | тост | `toast.*` (`shared/services/toast`) |
| Ошибка загрузки данных, у экрана есть свой UI | пустое состояние | `EmptyState` (+ «Повторить») |
| Опасное действие (удаление) | подтверждение | `ConfirmDialog` |
| Ещё не реализованная фича | карточка-заглушка | `SoonCard` |
| Непредвиденный краш рендера | fallback страницы | `ErrorBoundary` |

## Глобальный перехват (safety-net)

`QueryClient` настроен (`app/providers/queryClient.ts`): `QueryCache`/`MutationCache`
`onError` показывают `toast.error` **по умолчанию** для любого запроса/мутации. Текст —
из `ApiError` (сообщение бэка) или `texts.common.state.error`.

**Opt-out:** если экран показывает ошибку сам (форма через `FormNote` или список через
`EmptyState`), помечаем запрос/мутацию `meta: { silentError: true }` — иначе фидбэк
задублируется (и инлайн, и тост).

```ts
// форма: ошибку рисуем инлайн (FormNote) → глушим глобальный тост
useMutation({ mutationFn, meta: { silentError: true }, ... });

// запрос со своим EmptyState → глушим глобальный тост
useQuery({ queryKey, queryFn, meta: { silentError: true } });
```

## Правила выбора

1. **Форма (есть поля и submit)** → ошибки и успех **инлайн** через `FormNote`
   (`success` зелёная, `error` красная), мутация — `meta: { silentError: true }`.
   Примеры: вход/регистрация, смена email/пароля, переименование проекта.
   - Успех формы — **не тостом**, а `FormNote`. Тост у формы только если результат уводит
     с экрана и заметка не успеет прочитаться.
2. **Действие-кнопка без формы** (добавить/удалить язык, сменить роль, удалить раздел,
   добавить ключ, батч-сохранение) → **ошибка автоматически тостом** (глобальный обработчик,
   ничего не пишем вручную). **Успех** — `toast.success(...)` только если нет другого
   видимого подтверждения (напр. батч-сохранение: «Сохранено правок: N»). Если результат и
   так виден (строка исчезла/изменилась) — успех-тост не нужен.
3. **Загрузка данных** → если экран рисует свой `EmptyState`-error (таблица, языки,
   участники) — `meta: { silentError: true }` на запросе. Если выделенного UI нет — пусть
   срабатывает глобальный тост.
4. **Опасные действия** → всегда `ConfirmDialog` (danger), а не «тихое» выполнение.
5. **Не дублируем фидбэк**: один исход — один канал. Инлайн **или** тост, не оба.

## Стиль (токены)

- Ошибка: `--danger` / `--danger-tint` / `--danger-fg`. Успех: `--success*`. Инфо: `--info*`.
- Не вводить сырые hex и не верстать цветные блоки вручную — только `FormNote` / `Toast` /
  `EmptyState` / `SoonCard` (все уже на токенах, работают в light/dark).
- Тексты — только из `shared/resources/i18n` (общий фолбэк ошибки — `common.state.error`,
  «Скоро» — `common.state.soon`). Никаких строк-литералов в компонентах.

## Чек-лист при добавлении мутации/запроса

- [ ] Это форма? → `FormNote` инлайн + `meta.silentError` на мутации.
- [ ] Это действие-кнопка? → ошибку отдаём глобальному тосту; успех — `toast.success`
      только если изменение не видно глазом.
- [ ] Это запрос с собственным `EmptyState`? → `meta.silentError` на запросе.
- [ ] Опасное действие? → `ConfirmDialog`.
- [ ] Не появился ли двойной фидбэк (инлайн + тост)?
