# Запуск и публикация Stringly

Четыре конфигурации по двум осям: **где** работает (локально / на сервере) и **откуда данные**
(моки MSW / реальный бэк). Инструкция по каждой — в отдельном файле.

Здесь только инструкции (как работает, как запускать, как настроить). Рабочие файлы — скрипт
деплоя и конфиг nginx — лежат в `deploy/` в корне репозитория.

| # | Конфигурация | Где | Данные | Файл | Кратко |
|---|--------------|-----|--------|------|--------|
| 1 | Локал дев | локально | моки | [`01-local-dev-mock.md`](./01-local-dev-mock.md) | `npm run dev` |
| 2 | Локал прод | локально | реальный бэк | [`02-local-prod-real.md`](./02-local-prod-real.md) | `npm run start:local-real` |
| 3 | Демо дев | сервер | моки | [`03-demo-dev-mock.md`](./03-demo-dev-mock.md) | `./deploy/deploy.sh mock` |
| 4 | Демо прод | сервер | реальный бэк | [`04-demo-prod-real.md`](./04-demo-prod-real.md) | `./deploy/deploy.sh real` |
| — | Настройка сервера (разово) | сервер | — | [`05-server-setup.md`](./05-server-setup.md) | nginx + HTTPS |

## Как это устроено

Переключение делает **режим сборки Vite** (`--mode`) + соответствующий env-файл в корне:

| Режим (env-файл) | `VITE_API_MODE` | Используется в |
|------------------|-----------------|----------------|
| `.env.local-mock` | `mock` | конфиг 1 (`npm run dev`) |
| `.env.local-real` | `real` | конфиг 2 |
| `.env.demo-mock` | `mock` | конфиг 3 |
| `.env.demo-real` | `real` | конфиг 4 |

- `VITE_API_MODE=mock` → в `main.tsx` поднимается MSW и отдаёт данные **в браузере**; бэк не нужен.
- `VITE_API_MODE=real` → моки выключены, фронт шлёт запросы на `/api/v1` (относительный путь).
  Кто отвечает на `/api/v1`: **локально** — прокси Vite на бэк (`vite.config.ts`, по умолчанию
  `http://localhost:8080`, меняется переменной `BACKEND_ORIGIN`); **на сервере** — nginx-прокси
  (`deploy/nginx-stringly.conf`).
- Базовый адрес всегда относительный (`/api/v1`) — поэтому один и тот же код работает во всех
  четырёх конфигурациях без правок.

## Что нужно установить

- **Локальные конфиги (1, 2):** Node.js + `npm`. Для конфига 2 — запущенный Kotlin-бэк.
- **Серверные конфиги (3, 4):** на твоей машине — `npm`, `ssh`, `rsync`; на сервере —
  один раз настроенный nginx + самоподписанный сертификат (см. [`05-server-setup.md`](./05-server-setup.md)).

## Важно про моки и HTTPS

Моки (MSW) работают через Service Worker, а браузеры разрешают его только на **HTTPS или
localhost**. Поэтому конфиг 1 (localhost) — ок, а конфиг 3 (сервер) требует HTTPS
(используем самоподписанный сертификат по IP — см. файл конфига 3).
