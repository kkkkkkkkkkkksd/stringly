# Конфигурация 3 — Демо дев (моки на сервере)

Публикация на арендованный сервер демо-версии **на моках**. Бэкенд не нужен — данные живут в
браузере каждого посетителя. Самый быстрый способ показать сайт другим людям.

## Когда использовать

- Нужно дать ссылку на работающий фронт, пока бэка нет.
- Демонстрация UI/флоу без реальных данных.

## Разовая настройка сервера

Один раз настрой nginx + самоподписанный сертификат по инструкции
[`05-server-setup.md`](./05-server-setup.md).

## Команда (каждый деплой)

С твоей машины, из корня проекта:

```bash
./deploy/deploy.sh mock
```

(первый раз: `chmod +x deploy/deploy.sh`)

Открывать: **https://144.31.235.244.nip.io:8443** — первый раз браузер попросит подтвердить
самоподписанный сертификат («всё равно перейти»).

## Что происходит

- `deploy.sh mock` собирает фронт в режиме `demo-mock` (`.env.demo-mock`, `VITE_API_MODE=mock`)
  → в бандл попадает воркер MSW.
- `dist/` заливается на сервер через `rsync`, nginx раздаёт статику и перезагружается.
- В браузере посетителя стартует MSW и отдаёт моковые данные.

## Нюансы (важно)

- **Нужен HTTPS.** Service Worker (а значит MSW) браузеры разрешают только на HTTPS или
  localhost. Поэтому раздаём по `https://` с самоподписанным сертификатом — иначе демо будет
  без данных. Чистый сертификат без предупреждения — когда появится домен (см. `05-server-setup.md`).
- Данные у каждого посетителя свои и не сохраняются между перезагрузками (моки в памяти).
- Для реального многопользовательского входа/регистрации нужен бэк → конфигурация 4.
