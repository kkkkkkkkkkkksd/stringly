# Конфигурация 1 — Локал дев (моки)

Локальная разработка с горячей перезагрузкой. Данные отдаёт MSW в браузере, бэкенд не нужен.
Это основной режим для повседневной работы над фронтом.

## Когда использовать

- Пишешь/правишь UI и логику фронта.
- Бэка ещё нет или он не нужен.

## Команда

```bash
npm run dev
```

Откроется `http://localhost:5173` (Vite сам откроет браузер).

## Что происходит

- Vite поднимает dev-сервер с HMR (мгновенное обновление при правках).
- Режим `local-mock` → грузится `.env.local-mock` (`VITE_API_MODE=mock`).
- В `main.tsx` стартует MSW (`mockServiceWorker.js`) и перехватывает запросы на `/api/v1`,
  отдавая моковые данные из `src/mocks/handlers.ts`.

## Нюансы

- Service Worker на `localhost` разрешён, HTTPS не требуется.
- Данные не сохраняются между перезагрузками (моки в памяти) — это нормально для разработки.
- Если моки не срабатывают, проверь, что есть `public/mockServiceWorker.js`
  (генерируется `npm run mock:init`).
